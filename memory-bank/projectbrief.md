# プロジェクト概要 (projectbrief.md)

目的：様々なプラットフォームでカメラの状態情報を共有できる環境を提供する。

背景：Unity、UnrealEngine、Blenderなどプラットフォーム間でカメラの動作状態を共有する有効な手段が存在しなかった。

ゴール：
- プラットフォーム横断で使用可能なC++ DLLを開発
- DLLでカメラ位置、回転、ズーム等を収集し独自バイナリ形式で保存
- バイナリ形式で受信したデータを各プラットフォーム言語向けフォーマットに変換
- データ整合性のためCRC16チェック実装

スコープ：DLL自体の開発およびテスト、ドキュメント作成

制約条件：
- C++で開発し各プラットフォームAPIに依存しない設計
- DLLサイズ最小化、パフォーマンス最適化
- オープンソースライセンス採用（商用利用可）
- 自動化テストフレームワークによるユニット/統合テスト

リスク：
- プラットフォーム間互換性問題
- データ形式変更の既存コード影響
- セキュリティ脆弱性発生
